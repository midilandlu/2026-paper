# 透過 /teach 技能學習任何事物

了解如何使用我的 /teach 技能，它可以建立個人化課程、追蹤進度，並根據每位學習者的「近側發展區間（Zone of Proximal Development）」進行調整。非常適合用於新手引導（onboarding）與自主學習。

我前幾天突然意識到，我教東西已經有 10 年了。我當了 6 年的發聲教練，而教導開發者這份工作也做了 4 年。有一段時間我一直在想，如果能把我所知道關於教學的一切都放進一個「技能（skill）」裡，讓每個人都能學習任何事物，那該有多好。前幾天我在去倫敦的漫長巴士車程中，寫下了一個 teach 技能，結果發現它相當不錯。它教會了我如何解魔術方塊，你知道的，這是我一直想學但卻從未有時間或動力去做的事。[00:00:00 → 00:00:32]

但是有了這個 teach 技能，感覺就像是有了一位真實的老師，他用我喜歡的教學方式來教我，而且與我的目標完全一致。接下來，我會展示我是如何做到的。當我們在看一個教學技能時，關鍵的概念是：有些技能是「有狀態的（stateful）」，有些則是「無狀態的（stateless）」。如果一個技能是無狀態的，這意味著它不會保留過去執行的任何狀態，對你以前做過的事情也沒有任何記憶。[00:00:30 → 00:00:54]

換句話說，無狀態技能不會在本機檔案系統中儲存任何東西，來幫助它稍後從中斷的地方接續下去。而有狀態的技能則會這麼做。它們會將東西儲存到本機檔案系統，或是儲存到 MCP 伺服器中，留下它們稍後能追蹤的筆記。最初，我把 teach 想成是一種無狀態技能，你只需要對它說：「教我這個東西。」[00:00:54 → 00:01:16]

接著，它就會幫你找一些好的資源，並輸出一段關於那個主題的課程。但我意識到，我所有優質的教學都是有狀態的——當我（作為老師）教你的時候，我會記得你學到了哪裡，知道你學過什麼，了解你接下來可以學什麼，而且我腦中還有一堆過去教這個東西時收集來的好資源。因此，我決定 teach 必須是一個有狀態的技能。[00:01:16 → 00:01:41]

舉個例子，如果你以前用過我的 skills 儲存庫，你會知道 `grill me` 完全是無狀態的。它只會針對一個主題不斷考問你，直到你準備好去實作為止。但是，`grill with docs` 則是有狀態的。換句話說，它會在本機儲存一些 ADR（架構決策紀錄，Architectural Decision Records），也會在儲存庫中儲存一些詞彙表之類的東西。[00:01:41 → 00:02:03]

所以，`Grill with Docs` 是有狀態的，而且會隨著時間變得更好；而 `Grill me` 則是完全無狀態的。這兩者並沒有誰比誰好，它們只是在不同情況下各有用途。因此，當你在設計技能時，必須謹慎思考它們到底需要無狀態還是有狀態。而 Teach 需要是有狀態的。安裝這個技能的方法是，前往我的 skills 儲存庫，也就是 mapper（這邊應指他的 repo）。[00:02:03 → 00:02:24]

打開 repo，找到這裡的快速入門（quick start），然後直接執行 `skills.sh` 安裝程式，並選擇 Teach 技能。完成後，你進到一個空的目錄裡，然後在你的寫程式代理程式（coding agent）中執行 Teach。舉例來說，在這個也就是我放魔術方塊的目錄中，我說：「教我如何解魔術方塊。」[00:02:24 → 00:02:42]

你在我的學習過程中會注意到，它在這裡建立了多少檔案。不過它建立的第一個檔案是「任務目標（mission）」。我相信，要讓一位老師發揮效用，你需要了解學生「為什麼」想學這個東西。所以，我讓這個技能幫你建立一個任務目標。這裡寫著：「Matt 希望能夠在不借助外力的情況下，至少一次成功解開打亂的 3x3 魔術方塊。[00:02:42 → 00:03:03]

目標在於成就本身，而非速度，也不是理論。」接下來，它會建立一組資源（resources）。它會上網搜尋高可信度的實際主要來源（primary source），並將這些資源用來建立課程。它會在你第一次要求它教你某個東西時進行這項工作，然後在你學習的過程中持續更新。接著，它會為你建立第一堂課。[00:03:03 → 00:03:25]

這些課程會儲存在 `lessons` 資料夾中，並像這樣編號，它們全都是獨立的 HTML 檔案。HTML 真的比 markdown 豐富太多了，它能呈現更具表現力、互動性更高的內容。這正是讓這種學習體驗變得如此出色的核心所在。這是我的第一堂課：結構、符號和白色十字。它基本上只在那個當下教你需要知道的東西。[00:03:25 → 00:03:49]

它提供圖解、非常簡單的解說、重點提示（call outs），同時也給你測驗。在教任何東西時，我發現非常重要的一點就是要建立一個回饋迴圈（feedback loop），而測驗在這方面還算過得去。當你找不到更豐富的回饋迴圈時，測驗基本上是個不錯的選擇。同樣地，如果我們往下看，我們有符號說明、有容易犯錯的陷阱、還有你的第一項技能。所以，它不僅給我需要的知識，也鼓勵我發展我需要的技能，這兩者是略有不同的。[00:03:49 → 00:04:18]

而且這個技能本身知道這一點。當我完成課程，並說「是的，我可以拼出白色十字」時，它就會建議，或者說是記錄在這個學習紀錄（learning record）裡。當我回報學習狀況時，代理程式會建立這些非常簡單的紀錄。這能讓它持續追蹤我的學習進度，就像真正的老師一樣，為我量身打造下一堂課。這是這門課稍後段落的另一個解說。[00:04:18 → 00:04:47]

我想它算是在我們前進的過程中，根據我的需求創建出一門課。像這一個，它開始加入一些小小的引用來源。同樣地，還有更多測驗。我的意思是，我就是覺得這種排版看起來很舒服，也非常實用。我在打造這些內容時，也算是在一邊完善這個技能。[00:04:47 → 00:05:05]

所以，我在底部加入了這個小東西。它試圖幫你找到可以在這裡問問題的社群。因為沒錯，你可以在這裡培養知識與技能，但要對這件事培養出「智慧（wisdom）」，唯一的方法就是實際與社群互動，在現實世界中測試你的想法。它也會為你整理參考資料。我注意到這是我需要的東西。[00:05:05 → 00:05:27]

所以，這裡有個詞彙表（glossary）。這個詞彙表包含了結構、符號、解法握姿、小花圖案，以及任何我們學到的奇怪術語，都會被放進這個詞彙表中。這非常有用，特別是如果你正在用這個方法學習某種程式語言。這也意味著未來的課程可以更簡潔，因為它們可以直接參照詞彙表中的術語，而你在困惑時也可以回頭查閱。它還會為你製作小抄（cheat sheets）。[00:05:27 → 00:05:52]

所以這裡有一張解法卡（solve card），它基本上就是給了我完整的解法。如果我只想在一個單一的地方看看如何還原魔術方塊，這就是我可以去的地方。最後，稍微解釋一下這個 `notes.md` 檔案，這是代理程式可以記下我任何偏好、任何注意事項的地方，這純粹是代理程式本身的內部筆記。那麼，讓我們想像我坐下來開始一堂課，而且我基本上想讓老師了解自從我上次上課以來我做了些什麼。[00:05:52 → 00:06:22]

所以我只說了 teach，然後表示我現在基本上可以還原魔術方塊了，除了解角塊（corner cycle）那部分，我還沒有完全把它變成肌肉記憶。讓我們來看看它會說什麼。因為這是一個空的對話脈絡（context），所以它理所當然地從檢查我的教學工作區狀態開始。它檢查了所有的解法卡，看看我進展到哪裡。接著它對我的狀況給出了一個很不錯的診斷。[00:06:22 → 00:06:44]

它說解角塊（這是一種特定的演算法）的觀念很扎實，但就是還沒有達到肌肉記憶。讓我們先看一下之前的記憶課程，來配合原有的風格。所以我們基本上已經把代理程式變成了一位老師，而你必須像對待任何一對一老師那樣與它合作。你告訴它你遇到什麼問題，它就會設計課程來尋找解決方案。對了，我在這裡使用的模型是 Opus 4。[00:06:44 → 00:07:09]

用了中等的處理強度。這裡它也使用了一個關鍵的教學術語：「近側發展區間」。一個演算法，純粹的肌肉記憶訓練。近側發展區間是一個我非常著迷的關鍵教學理念。這個理念是說，你應該永遠在能給學生完美挑戰但又不至於讓他們感到畏懼的範圍內進行教學。[00:07:09 → 00:07:28]

這意味著每一堂課都必須簡潔、緊湊，並且準確地設定在那個近側發展區間內。這樣學生既不會無聊，也不會被嚇壞。所以我們可以看到它現在建立了這堂課——解角塊課程。它為我們精準打造了這堂課。所以，它基本上就是把它拆解，更專注在上面，給我新的心智模型。[00:07:28 → 00:07:49]
